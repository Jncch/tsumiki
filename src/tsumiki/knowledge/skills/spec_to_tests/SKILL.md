# spec_to_tests — 仕様書 → テストケース生成ガイドライン

## テストケース 1 件の構成

Given / When / Then 形式 (BDD スタイル):

- **Given** (前提): 機能の入力条件 / システム状態
- **When** (操作): ユーザーアクション / API call
- **Then** (期待): 期待される出力 / 状態遷移

## 必須キーワード

- "Given", "When", "Then" を全て含む
- 仕様書中の機能名 (見出し) を最低 1 つ含む

## 文字数の目安

- 1 テストケース: 100〜500 文字
- 複数テストの場合は番号付きリスト

## トーン

- 簡潔、命令形 (検証スクリプト相当)
- 具体例で説明 (抽象論を避ける)

## 禁止表現

- "適切に動作する" のような曖昧な期待値
- TODO / FIXME を残す
